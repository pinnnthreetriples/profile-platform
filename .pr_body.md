# Stage 2 ‚Äî Profiles

## Summary

Implements a full user profile system on top of the Stage 1 Supabase Auth foundation.
Users can view and update their display name, bio, and avatar URL. Payment status is
read-only and controlled server-side only.

## Scope

- Supabase migration: `profiles` table with constraints and RLS
- Profile feature module: schemas, types, server functions, server action, queries,
  components
- `/profile` page: real server-side data, replaces placeholder
- Unit tests: schemas, server functions, actions
- E2E: unauthenticated redirect verified
- Documentation: README, TODO, PROFILE_FLOW.md, STAGE_2_COMPLETE.md

**Not included:** payments, BTCPay, webhooks, service role on frontend, UI polish.

## DB Migration

File: `supabase/migrations/20260515000000_create_profiles.sql`

Table `public.profiles`:

- `id uuid` ‚Äî PK, references `auth.users(id)` ON DELETE CASCADE
- `display_name text` ‚Äî nullable, max 80 chars (DB constraint)
- `bio text` ‚Äî nullable, max 500 chars (DB constraint)
- `avatar_url text` ‚Äî nullable
- `payment_status text` ‚Äî not null, default `'pending'`, enum constraint
  (`pending|paid|failed|expired|cancelled`)
- `created_at`, `updated_at` ‚Äî auto-managed; `updated_at` via trigger

## RLS Policies

| Policy                       | Operation | Rule                                      |
| ---------------------------- | --------- | ----------------------------------------- |
| Users can read own profile   | SELECT    | `auth.uid() = id`                         |
| Users can insert own profile | INSERT    | `auth.uid() = id`                         |
| Users can update own profile | UPDATE    | `auth.uid() = id` (using + with check)    |

No public read. No cross-user access.

## Security Notes

- `server.ts` starts with `import "server-only"` ‚Äî never importable in client code
- All DB operations use anon SSR client + RLS ‚Äî no service role
- `payment_status` is not in the update schema ‚Äî cannot be changed via profile form
- `getUser()` used for server-side auth checks (not `getSession()`)
- No secrets in code
